Проблема

Три барьера блокируют AI в инвестиционной компании

60-80% данных, с которыми работают аналитики, содержат ПДн клиентов. Это делает использование внешних AI-сервисов юридически невозможным без технической защиты.

01
Регуляторный
Штрафы до 500 млн руб. и уголовная ответственность
ФЗ-152 запрещает передачу ПДн третьим лицам. Требования ЦБ РФ (683-П, 684-П) ужесточают контроль. Каждый запрос к внешнему LLM с клиентскими данными — потенциальное нарушение.
02
Организационный
CISO блокирует все AI-инициативы
У службы безопасности нет технического инструмента, позволяющего дать формальное разрешение на использование AI с клиентскими данными. Результат: все проекты заморожены на согласовании.
03
Конкурентный
Финтехи уходят вперёд, сотрудники — в Shadow AI
Конкуренты уже используют AI для анализа и клиентского сервиса. Сотрудники используют ChatGPT с личных устройств — без контроля, аудита и защиты данных.
Как работает

Прозрачный прокси между приложением и LLM

Сервис с OpenAI-совместимым API. Подключение — смена одного URL, без изменения кода приложений.

Шаг 1
Запрос сотрудника
Текст с ФИО, ИНН, номерами счетов клиентов
Шаг 2
Обнаружение ПДн
Три уровня детекции: нейросеть + русский NER + regex
Шаг 3
Маскирование
ПДн заменяются на токены. Маппинг сохраняется в Redis
Шаг 4
Запрос в LLM
Только обезличенный текст покидает периметр
Шаг 5
Обратная замена
Токены заменяются на реальные данные в потоке ответа
Сотрудник видит полный ответ с реальными данными. LLM — никогда.
Персональные данные не сохраняются в модуле: маппинг в Redis удаляется автоматически после обработки запроса (TTL). Ответ приходит в потоковом режиме — без ожидания полной генерации.
Архитектура

Трёхуровневая детекция: defense in depth

Ни одна модель NER не даёт 100% полноты. Три независимых уровня перекрывают слепые зоны друг друга.

Уровень 1 — GPU
Нейросеть GLiNER
Zero-shot NER: распознаёт ФИО, адреса, организации, произвольные типы сущностей. Новые категории данных добавляются через конфигурацию без переобучения модели. Multilingual, работает с русским языком.
Уровень 2 — CPU
Русский NER (Natasha)
Специализированная модель для русского языка. Высокая точность на именах, локациях и организациях. Компактная (~30MB), работает на CPU. F1 близок к state-of-the-art.
Уровень 3 — Regex
Детерминистичные правила
100% точность на структурированных форматах: серия/номер паспорта РФ, ИНН (10 и 12 знаков), СНИЛС, номера банковских карт и счетов, телефоны, email. Расширяются под специфику заказчика.

Compliance

Что получает CISO

Техническое обоснование для формального разрешения AI. Не бумажное, а инженерное.

Требование Как закрыто
ФЗ-152: запрет передачи ПДн ПДн маскируются до отправки. LLM получает только обезличенный текст. Передачи персональных данных третьим лицам не происходит
ЦБ РФ 683-П / 684-П On-premise развёртывание, данные в периметре. Маппинг с TTL, нет persistent storage ПДн. Соответствие требованиям к защите информации в финансовых организациях
Аудит ИБ Логирование метаданных: timestamp, user ID, количество и типы замаскированных сущностей. Полный аудит-трейл без хранения самих ПДн
Контроль Shadow AI Единая точка входа для всех AI-запросов. Сотрудники работают через корпоративный интерфейс с автоматической защитой, а не через личные устройства
Личная ответственность CISO/DPO Формальное техническое обоснование: «ПДн защищены автоматическим маскированием, что подтверждается логами системы». Основание для разрешения AI-проектов
